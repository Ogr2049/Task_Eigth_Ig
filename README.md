Описание: Микросервисная платформа для управления пользователями с автоматической отправкой уведомлений по электронной почте.  Система состоит из нескольких взаимосвязанных микросервисов, работающих в контейнерной среде Docker.

1)User Service (user-service-module)
  
Назначение: Управление пользователями (CRUD операции)

Функциональность: 

-cоздание, чтение, обновление и удаление пользователей

-Валидация данных пользователя

-Отправка событий в Kafka при создании/удалении пользователя

-REST API с Swagger документацией

-Поддержка HATEOAS для навигации по API


2)Notification Service (notification-service):

Назначение: отправка email-уведомлений пользователям.

Функциональность:


-Получение событий из Kafka о создании/удалении пользователей

-Отправка приветственных писем при создании аккаунта

-Отправка писем об удалении аккаунта

-REST API для прямой отправки email

-Интеграция с SMTP сервером для отправки реальных писем


Вкратце как работает notification-service:
Код использует Spring Mail для отправки email через SMTP протокол. В конфигурации (application.yml) указаны параметры SMTP сервера. Для работы нужен реальный или тестовый SMTP сервер (например MailHog). Для тестов при отправке писем желательно использовать MailHog. 

Как проверить работу email:

MailHog - тестстовый SMTP сервер, который перехватывает письма и показывает их в веб-интерфейсе.


1)Добавить MailHog в docker-compose.yml:


mailhog:
  
	image: mailhog/mailhog
  
	container_name: mailhog
  
	ports:
    
		- "1025:1025"  # SMTP порт
    
		- "8025:8025"  # Веб-интерфейс
  
	networks:
    
		- user-service-network


2)Обновить конфигурацию notification-service:


spring:
  
	mail:
    
		host: mailhog
    
		port: 1025
    
		username: ''
    
		password: ''


3)Можно просмотреть письма в веб-интерфейсе: http://localhost:8025
